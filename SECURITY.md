# Security Policy

## Reporting a Vulnerability

Imagineer Mode is a cognitive specification, not executable software. It contains no code, no dependencies, and no attack surface in the traditional sense.

If you discover a way the specification produces harmful, manipulative, or unsafe AI behavior, please report it responsibly:

**Email:** attila@livingmirrors.ai

**What to include:**
- The AI model tested
- The exact system prompt used
- The input that triggered the issue
- The problematic output
- Why you consider it harmful

**Response time:** Within 48 hours.

**What counts as a vulnerability:**
- The specification causing an AI to bypass safety guidelines
- Principles that produce manipulative or deceptive output
- Output layers that could be used to mislead users
- Any interaction pattern that causes psychological harm

**What does not count:**
- The AI not following the specification perfectly (that's a limitation, not a vulnerability)
- Disagreement with the principles (open an issue instead)
