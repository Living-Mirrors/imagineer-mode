# Contributing

*How to build your own process mode.*

---

## The Idea

Imagineer Mode is one process specification: construction through experience design. It was built from the methodology of Walt Disney Imagineering cross-referenced against the neuroscience of constructive imagination.

But Imagineering is not the only process architecture. Design thinking, biomimicry, agile, systems design, speculative design, permaculture - each has a cognitive framework that structures how new things come into existence. Each one could be a process mode.

This is an invitation to build yours.

---

## Structure of a Process Mode

Every process mode follows the same pattern:

### 1. Name and Axis

The mode name and its axis position.

```
Mode: [Name] Mode
Axis: Process (Axis III)
```

Example: *Imagineer Mode - Axis III, The Process*

### 2. Principles

The core construction patterns. These should be:

- **Grounded in methodology AND neuroscience.** Not preferences. Not habits. Actual process patterns with research backing.
- **Ordered from most structured to most expansive.** The progression mirrors the dial from analytical to immersive.
- **Observable.** You should be able to point at a response and say "that principle is active" or "that principle is not."
- **Between five and nine.** Fewer than five is too coarse. More than nine is too granular for an LLM to hold distinctly.

### 3. Output Layers

What the process makes visible. These should be:

- **Process-focused.** They shape how things get built, not how information looks or how the interaction flows.
- **Distinct from lens and dynamic layers.** If a layer duplicates what creative/savant or play already does, it belongs on a different axis.
- **Marked.** Each layer has a visible marker so the user can see which layer is active.

### 4. The Dial

A continuous 0-100 scale. What is active at 30% vs 70% vs 100%. The dial keeps the mode usable across contexts.

### 5. Origin

Where this comes from. The methodology. The research. The personal experience. Why this process matters and who it serves.

## How to Contribute

### Option 1: Build from your own practice

If you have a construction methodology that produces consistently exceptional work, you already have the source material. The question is: what does your process do that default analytical construction does not? What principles drive it? What becomes visible that was hidden?

### Option 2: Build from research

Pick a construction methodology with documented research. Read the literature. Extract the process patterns. Build the spec. Test it.

### Option 3: Improve Imagineer Mode

Found a principle that doesn't produce observable change? An output layer that doesn't earn its place? Open an issue. Or fix it and submit a PR.

---

## Guidelines

- **Be honest.** If a principle does not change the construction process in observable ways, it is not a principle. It is a wish.
- **Be specific.** "Build creatively" is not a principle. "Start from the desired experience, not the thing to build" is.
- **Be practical.** Every principle should change what the AI actually produces.
- **Test before submitting.** Run your spec against multiple AI models. Does the output actually change?
